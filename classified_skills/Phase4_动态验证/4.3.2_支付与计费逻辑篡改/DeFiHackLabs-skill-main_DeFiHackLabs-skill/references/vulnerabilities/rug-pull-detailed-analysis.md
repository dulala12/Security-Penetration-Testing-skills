# Rug Pull 详细分析

项目方恶意或半恶意退出导致的损失场景。

## 主要子类型

- **Liquidity Removal**: 抽走流动性
- **Backdoor Function**: 隐藏的管理后门
- **Mint Abuse**: 超发或无限铸币

## 典型攻击路径

1. 选择可操纵的输入/状态（价格源、回调、参数）
2. 放大可利用窗口（组合闪电贷、时序缺陷或权限绕过）
3. 触发漏洞获得不当收益
4. 清算或洗出资金

## 检测清单

- 检查管理权限过大
- 检查流动性锁定与释放规则
- 检查铸币与手续费参数

## 防御要点

### P0 (必须)
- 多签与时间锁保护管理操作
- 锁定流动性并公开规则

### P1 (推荐)
- 公开治理变更流程
- 增加合约监控与报警
