# Logic Flaw 详细分析

算法实现与条件判断错误导致的漏洞类型。

## 主要子类型

- **Calculation Error**: 计算错误或顺序错误
- **Condition Bypass**: 关键条件可被绕过
- **State Sync Error**: 状态同步或缓存过期

## 典型攻击路径

1. 选择可操纵的输入/状态（价格源、回调、参数）
2. 放大可利用窗口（组合闪电贷、时序缺陷或权限绕过）
3. 触发漏洞获得不当收益
4. 清算或洗出资金

## 检测清单

- 检查关键分支条件是否可绕过
- 检查计算顺序与精度
- 检查缓存/状态更新时间

## 防御要点

### P0 (必须)
- 关键分支增加断言与测试
- 复核边界条件与极端值

### P1 (推荐)
- 复杂计算拆分并加入中间检查
- 引入状态版本或时间戳保护
