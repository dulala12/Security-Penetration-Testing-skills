# 语义陷阱与抗幻觉护栏（Java审计）

## 用词约束

优先使用边界清晰词：

- 用“漏洞/缺陷/错误”替代泛化的“风险/问题/异常”
- 用“检查/列出”替代“分析/审查”
- 用“要求”替代“建议”

## 结论分级

统一输出：

- `CONFIRMED`：证据链完整且可复现
- `SUSPECTED`：证据链部分成立，需补证据
- `INFO`：历史线索或安全建议

禁止只给“高风险/低风险”而无证据链。

## 范围控制

仅评估本 skill 定义的漏洞类型，不扩展到：

- 代码风格
- 性能优化
- 与安全无关的架构偏好

## 输出硬约束（8字段）

1. 位置（文件/类/方法）
2. 参数（可控输入）
3. 漏洞类型
4. 证据链（source -> sink）
5. 防护状态（缺失/可绕过/有效）
6. 结论分级（CONFIRMED/SUSPECTED/INFO）
7. 修复建议
8. 复测步骤

缺字段则无效，必须重做。

## 自检

- 排除测试：是否出现范围外结论
- 最小对比：无漏洞样例是否过度输出
- 换词对照：出现发散时替换更窄语义词
