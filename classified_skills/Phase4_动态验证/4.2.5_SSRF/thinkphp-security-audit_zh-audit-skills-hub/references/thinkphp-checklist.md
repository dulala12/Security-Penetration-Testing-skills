# ThinkPHP 审计清单（TP3/TP5/TP6）

## 1) 路由与入口
- 是否存在调试/测试路由未下线。
- 是否存在可绕过的路由规则（大小写、模块映射、别名）。
- 是否直接暴露高权限后台接口。

## 2) 控制器输入处理
- 参数是否统一校验（类型/长度/范围/语义）。
- 是否直接信任 `$_GET/$_POST/Request::param()`。
- 是否将参数直接透传到模型层和模板层。

## 3) 数据库操作
- 是否拼接 where/order/limit/group 片段。
- 是否使用 `Db::query/Db::execute` 直接执行拼接 SQL。
- 排序字段是否做白名单。

## 4) 文件上传/下载
- 上传是否同时校验扩展名、MIME、魔数。
- 上传路径是否可控、是否在 Web 可执行目录。
- 下载接口路径是否可控，是否存在 `../` 穿越。

## 5) 模板渲染
- 是否把用户输入作为模板片段参与渲染。
- 是否有危险标签/表达式执行路径。

## 6) 命令执行/代码执行
- 检查 `eval/assert/system/exec/passthru/shell_exec/proc_open/popen`。
- 检查动态调用（`call_user_func`、反射）是否可控。

## 7) 反序列化
- 是否存在 `unserialize` 处理不可信数据。
- 是否存在可触发魔术方法链。

## 8) 鉴权与越权
- 管理接口是否统一鉴权中间件。
- 是否仅前端控制权限。
- 是否校验资源归属（user_id/tenant_id）。

## 9) 敏感信息与配置
- `.env`、配置文件、密钥是否泄露。
- 错误堆栈是否对外暴露。
- 调试模式是否在生产开启。
