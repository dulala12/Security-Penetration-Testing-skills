# ThinkPHP 二次开发高频坑

## 参数处理
- 为了省事关闭验证器或仅做前端校验。
- 将“字符串参数”直接作为 SQL 条件或排序字段。

## 权限设计
- 控制器遗漏鉴权中间件。
- 只校验登录，不校验角色与资源归属。

## 文件能力
- 上传文件直接使用原始文件名。
- 下载接口未做路径规范化。
- 上传目录配置在可执行路径下。

## 调试残留
- 测试接口、批处理接口、导入导出调试功能上线未关闭。
- 把异常堆栈直接返回前端。

## 工具类复用风险
- 老代码中的危险函数被业务接口间接调用。
- 底层 HTTP 客户端未做 SSRF 限制。

## 审计建议
- 对“快速修 bug”类提交重点审计。
- 对“权限、上传、导入导出、报表查询”模块做专项复查。
