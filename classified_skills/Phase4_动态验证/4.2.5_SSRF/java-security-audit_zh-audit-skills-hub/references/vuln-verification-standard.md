# 漏洞确认与复测标准

## 证据级别

- L1（线索）：仅关键词命中，无完整链路。
- L2（可疑）：链路部分可达，但缺关键条件。
- L3（确认）：可稳定触发，具备利用条件。

## 输出时必须区分

- 已确认漏洞（L3）
- 可疑点（L2）
- 审计建议（L1）

## 复测最小闭环

1. 修复前：记录触发条件与现象。
2. 修复后：相同输入应无法触发。
3. 回归测试：正常业务流不受影响。
4. 旁路测试：相邻接口/参数不能绕过。

## 修复优先级建议

- P0：可直接利用、影响核心资产。
- P1：可利用但条件较多或影响局部核心业务。
- P2：低可利用或影响较小。
- P3：规范性改进。
