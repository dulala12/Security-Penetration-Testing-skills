# RuoYi Nday 复盘方法

## 目标

把“历史漏洞信息”转为“当前版本可验证结论”。

## 方法

1. 收集历史漏洞条目（来源、时间、影响版本）。
2. 抽象根因（鉴权缺失、参数拼接、路径控制、配置问题）。
3. 在当前代码中按根因搜索同类实现。
4. 标注补丁状态：已修复 / 部分修复 / 未修复 / 无法确认。
5. 输出证据链与复测建议。

## 高频接口族（来自若依 Nday 实战补充）

优先关注后台用户管理接口族（常见于 `system/user/*`）：

- 用户查询、列表、导出、详情、增删改、角色关联、重置密码等接口
- 这类接口常同时涉及：权限校验 + 数据权限 + 关键参数传递

审计动作：

- 核对每个接口的鉴权注解与数据权限注解是否完整
- 检查是否存在“只校验登录，不校验资源归属”的逻辑
- 检查查询参数、排序字段、批量参数是否可控并进入 SQL 层

## 内部因子（开发实现侧）

针对文章中的“内部因子”思路，审计时增加两条：

- 从 `src/` 代码层面回溯接口实现，避免只看路由和控制器声明
- 对看似相似的接口做差异比对（同类接口不一定共享同一防护）
