# 用友漏洞族检查清单（详细）

## 1) 反序列化

检查点：
- 是否存在对象流直接读取（含 gzip 包裹对象流）。
- 是否有可到达的 `ObjectInputStream`/自定义流包装。
- 是否存在可从网络输入直达反序列化入口。

证据建议：
- 记录入口接口、读取函数、异常/行为侧证。
- 标注是否需要特定 gadget 前提。

## 2) SQL 注入

检查点：
- Action/Servlet 参数是否进入字符串拼接 SQL。
- 动态条件（排序、过滤、ID 列表）是否白名单。
- 是否出现延时盲注/报错注入特征。

证据建议：
- 记录参数名、拼接点、执行函数。
- 将“疑似延时”与“稳定可复现”分开标注。

## 3) XXE / XML 注入

检查点：
- XML 解析工厂是否禁用了外部实体。
- SOAP/XML 接口是否允许不可信 XML 直接解析。
- gzip/XML 组合解析链是否存在旁路。

证据建议：
- 给出解析配置与调用链证据。

## 4) 文件上传

检查点：
- 文件名/路径是否可控。
- 类型校验是否仅看后缀或 Content-Type。
- 上传目录是否可直接访问执行。

证据建议：
- 上传响应 + 落地可访问证据。
- 分清“上传成功但不可执行”与“可执行落地”。

## 5) 文件读取/路径穿越

检查点：
- 下载接口是否可控路径。
- base64 路径参数是否缺规范化校验。
- 是否可越过基准目录。

## 6) 未授权访问/鉴权绕过

检查点：
- 规则顺序导致的放行覆盖。
- 后缀绕过（如静态资源后缀伪装）
- 仅前端鉴权、后端无资源归属校验。
