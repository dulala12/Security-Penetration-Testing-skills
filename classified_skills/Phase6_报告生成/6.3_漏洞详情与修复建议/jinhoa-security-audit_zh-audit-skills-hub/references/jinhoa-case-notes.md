# 金和 OA（C6）审计要点参考

> 来源：你提供的《从金和OA C6学习C#代码审计》文档提炼。

## 关键经验

- 审计入口优先看 `aspx/asmx`，尤其是移动端服务接口。
- 从参数名入手追踪到 SQL 语句构造点（例如 userID、登录编码等）。
- 注意“功能链复用”：同一功能在多个页面/编号页面重复调用。
- 对“未授权 + 路径绕过 + 上传/下载”组合链重点验证。

## 高频漏洞线索（文档中出现）

- SQL 注入（多处 asmx/aspx 入口，拼接 SQL 或格式化拼接）
- 任意文件上传与落地访问
- 任意文件下载/路径读取
- 未授权访问
- XXE 注入

## 审计动作建议

1. 收集所有 `asmx` 服务方法及参数。
2. grep `SELECT/UPDATE/DELETE` 与 `string.Format`/拼接语句。
3. 核查 `Session`/登录态是否在每个敏感方法里强制校验。
4. 核查文件上传点是否限制后缀、内容和落地目录。
5. 核查 XML 处理逻辑是否关闭外部实体。

## 输出注意

- 将“线索”与“确认漏洞”分开。
- 标注版本与路径，避免跨版本误判。
- 对不可稳定复现的问题标为“可疑点待验证”。
