# 若依 Nday 常见模式

> 基于你提供的材料整理，作为审计“线索入口”，不是直接漏洞结论。

## 1) 弱口令与历史口令复用

常见现象：

- 管理员口令弱（如简单数字/默认组合）
- 登录页存在历史版本口令残留可用

审计动作：

- 核查默认账号策略、首次登录强制改密、口令复杂度策略
- 核查是否存在测试账号、演示账号、硬编码口令
- 核查失败次数限制、验证码、二次验证是否生效

## 2) SQL 注入高频位点（示例）

截图中出现的高频点思路：

- `/system/role/list` 中 `params[dataScope]` 一类参数
- `/system/dept/edit` 中 `ancestors` 一类层级字段

审计动作：

- 检查 Controller 参数到 Mapper/XML 的链路
- 检查 `${}` 与动态拼接 where/order 的可控性
- 检查字段白名单、参数化和数据权限过滤是否生效

## 3) Druid 页面暴露

常见路径：

- `/druid/index.html`
- `/druid/login.html`
- 以及 `prod-api/dev-api/admin-api` 等前缀组合路径

审计动作：

- 核查 Druid 是否对公网暴露
- 核查账号口令与 IP 白名单
- 核查是否关闭敏感 SQL/数据源细节外显

## 4) Swagger / OpenAPI 暴露

常见线索：

- `/v3/api-docs`
- Swagger UI 可直接浏览接口

审计动作：

- 核查生产环境是否禁用文档接口
- 核查是否存在未鉴权调试接口
- 核查文档是否泄露内部路径、参数与模型结构

## 5) Shiro 相关风险线索

常见线索：

- rememberMe / key 配置不当
- 历史配置与当前版本混用

审计动作：

- 核查 Shiro 版本、配置与密钥管理
- 核查 rememberMe 是否最小化使用并安全配置
- 核查会话固定、伪造与权限绕过防护

## 6) 敏感接口族清单化排查

截图中显示大量 `system/*` 路径，建议按“接口族”批量审计：

- 用户、角色、部门、租户、消息、文件、权限规则
- 增删改查、导入导出、批量操作、重置密码

审计动作：

- 逐接口核查鉴权注解 + 数据权限 + 资源归属
- 逐接口核查可控参数是否进入危险 sink
- 对“批量接口”增加越权和注入专项复测
