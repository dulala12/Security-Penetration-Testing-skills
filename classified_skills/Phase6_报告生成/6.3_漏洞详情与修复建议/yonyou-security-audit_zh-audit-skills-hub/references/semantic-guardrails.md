# 语义陷阱与抗幻觉护栏（用友审计）

## 用词约束

优先使用边界清晰词：

- 用“漏洞/缺陷/错误”替代“风险/问题/异常”
- 用“检查/列出”替代“分析/审查”
- 用“要求”替代“建议”

## 结论分级

统一输出：

- `CONFIRMED`：有稳定证据链
- `SUSPECTED`：证据不足需补充
- `INFO`：历史或建议性信息

## 范围控制

只评估当前 skill 定义的漏洞族；不将以下内容直接判漏洞：

- 历史文章中的未验证结论
- 纯性能/代码风格问题
- 未形成可利用链路的泛安全建议

## 输出硬约束（8字段）

1. 产品线/版本
2. 位置或接口
3. 参数（可控输入）
4. 漏洞类型
5. 证据链（source -> sink）
6. 结论分级（CONFIRMED/SUSPECTED/INFO）
7. 修复建议
8. 复测步骤

## 自检

- 排除测试：是否越界
- 最小对比：无漏洞样例不过度输出
- 换词对照：宽词替换窄词后复核
